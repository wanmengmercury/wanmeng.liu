Cincinnati, OH

March 2025 – Present

  • Built a simple frontend and DCM processing modules in the backend. The backend module takes a folder of DICOM files with missing metadata fields and a reference DICOM, copies values of tags such as study info, modality, institution info, and patient info from the reference file to the DICOM files in the folder, and assigns new values for the series UID, series description, series number, and instance number to the DICOM files in the folder.
  • The app can also take a folder of JPGs and a reference DICOM file, convert the JPGs to DCM first, and then go through the same processing steps as above.
  • The app can also take a video file, convert it to JPG, then convert it to DCM, and go through the same processing steps as above.
  • The user can download the processed DCM files in a zip file, which is later used to link to an existing patient study in the Picture Archiving and Communication System (PACS).
  • Deployed the app to the local machines of the physicians at the Heart Institute, where it is used daily.
